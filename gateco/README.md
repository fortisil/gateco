## Prerequisites

- Node.js 18.0 or higher

- Python 3.9 or higher
- pip (Python package manager)
- Docker and Docker Compose (recommended for local development)

## Installation

```bash
cd gateco

# Install all workspace dependencies
npm install

# Install backend Python dependencies
cd apps/backend && pip install -r requirements.txt && cd ../..

# Install website dependencies (if not covered by workspace)
cd apps/website && npm install && cd ../..
```

## Environment Setup

1. Copy the example environment file:
   ```bash
   cp .env.example .env
   ```

## Running the Application

### Run Everything (Docker Compose)

```bash
docker-compose up
```

### Run Individual Apps

#### Frontend (React/Vite)

```bash
cd apps/frontend
npm run dev      # Development server
npm run build    # Production build
npm test         # Run tests
```

#### Backend (Python/FastAPI)

```bash
cd apps/backend
uvicorn src.backend.main:app --reload --port 8000   # Development server
pip install -e .                                      # Build/install
pytest tests/ -v                                      # Run tests
```

#### Website (Next.js)

```bash
cd apps/website
npm run dev      # Development server (port 3001)
npm run build    # Production build
npm test         # Run tests
```

### Run All Tests

```bash
npm run test:all
```

### Build All

```bash
npm run build:all
```

## Deployment

Each app can be built and deployed independently as a Docker container.

### Build Docker Images

```bash
# Frontend
docker build -t gateco-frontend -f apps/frontend/Dockerfile apps/frontend

# Backend
docker build -t gateco-backend -f apps/backend/Dockerfile apps/backend

# Website
docker build -t gateco-website -f apps/website/Dockerfile apps/website
```

### Deploy with Docker Compose

```bash
# Start all services
docker-compose up -d

# View logs
docker-compose logs -f

# Stop all services
docker-compose down
```

### Individual App Deployment

#### Frontend (Static Site)
The frontend builds to static files that can be served by any CDN or static hosting:
```bash
cd apps/frontend
npm run build
# Deploy the dist/ folder to your hosting provider (Vercel, Netlify, S3, etc.)
```

#### Backend (API Server)
The backend runs as a Python ASGI server:
```bash
cd apps/backend
pip install -r requirements.txt
uvicorn src.backend.main:app --host 0.0.0.0 --port 8000
```

#### Website (Next.js SSR/SSG)
The website can be deployed as a Node.js server or exported as static files:
```bash
cd apps/website
npm run build
npm start                # Run as Node.js server
# Or deploy to Vercel/Netlify for automatic SSR support
```
